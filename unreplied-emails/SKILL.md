# 未返信メール抽出スキル

受信トレイから、稲垣さん（yinagaki@crooz.co.jp）が未返信のメールだけを正確に抽出し、蓄積リストを管理するスキル。

## 状態ファイル

以下のファイルで状態を永続管理する:

- `C:\Users\yinagaki\.claude\skills\unreplied-emails\state.json`

state.json の構造:
```json
{
  "last_run_at": "2026-03-30T11:00:00+09:00",
  "unreplied_emails": [
    {
      "no": 1,
      "thread_id": "...",
      "sender": "...",
      "subject": "...",
      "body": "...",
      "received_at": "2026-03-30T10:00:00+09:00"
    }
  ]
}
```

ファイルが存在しない場合は `last_run_at: null`、`unreplied_emails: []` として扱う。

## モード判定

ユーザーのリクエストに応じて以下のモードを使い分ける:

- **チェックモード**（「未返信メール確認して」「対応漏れある？」等）→ Step 1〜5 を実行
- **削除モード**（「No.2 は返信不要」「No.1 消して」等）→ Step 6 のみ実行
- **表示モード**（「ボタンで操作したい」「UIを開いて」「一覧を開いて」等）→ Step 7 のみ実行

---

## チェックモード: Step 1〜5

### Step 1: 状態ファイルの読み込み

`C:\Users\yinagaki\.claude\skills\unreplied-emails\state.json` を Read ツールで読み込む。

- `last_run_at` を取得する（nullの場合は7日前として扱う）
- `unreplied_emails` を既存の蓄積リストとして保持する

### Step 2: 候補メールの検索（差分のみ）

`gmail_search_messages` で以下の条件で検索する:

```
is:inbox to:yinagaki@crooz.co.jp -category:promotions -category:updates -category:forums -from:noreply -from:no-reply -from:notification newer_than:7d
```

取得したメッセージのうち、**受信日時が `last_run_at` 以降のものだけ**を処理対象とする。
すでに蓄積リストに同じ `thread_id` が存在するものもスキップする。

### Step 3: 各スレッドの返信状況を確認（最重要ステップ）

処理対象スレッドの**各スレッド**について `gmail_read_thread` で全メッセージを取得し、以下を確認する（並列処理推奨）:

- スレッド内に `From: "稲垣佑介" <yinagaki@crooz.co.jp>` のメッセージが存在するか
- 稲垣さんのメッセージが存在する場合 → **返信済み（除外）**
- 稲垣さんのメッセージが存在しない場合 → **未返信（候補）**

未返信と判定したスレッドについて、**先方の最新メッセージのID**（`message_id`）を取得する。
これはスレッド内の最後のメッセージ（稲垣さん以外）のIDを使う。

この確認を省略してはならない。

### Step 4: フィルタリング

以下のメールは除外する:

- BCC一斉送信（To が undisclosed-recipients のもの）
- 稲垣さんが Cc のみで、主な宛先が別の人のメール（稲垣さんへのアクションが明示的に求められている場合は含める）
- 自動送信・システム通知メール
- 先方からの「確認しました」「承知しました」等の完了報告で、追加の返信が不要なもの

### Step 5: 状態ファイルの更新と結果表示

1. 新たに見つかった未返信メールを蓄積リストに追加する
   - `no` は蓄積リスト全体で連番になるよう採番する（既存の最大No.+1から始める）
   - `message_id` フィールドに先方最新メッセージのIDを保存する（Gmail直リンク用）
2. `last_run_at` を現在時刻（JST）に更新する
3. 更新した state.json を Write ツールで保存する
4. 蓄積リスト全件を以下の形式で表示する:

```
---
【No.】1
【送信者】○○株式会社 △△ <email@example.com>
【件名】○○○○
【受信日】2026-03-30
【本文】
（元メールの本文をそのまま記載する。ただし引用部分・署名ブロックは省略する）
---
```

表示ルール:
- 蓄積リスト全件を表示する（新着分だけでなく過去分も含む）
- 本文は省略せず全文を記載する（引用部分「> 」で始まる行と署名ブロックのみ省略）
- 件数が多い場合でも省略しない

---

## 削除モード: Step 6

ユーザーが「No.X は返信不要」「No.X 消して」等と指示した場合:

1. state.json を Read ツールで読み込む
2. 指定された No. のエントリを `unreplied_emails` から削除する
3. 更新した state.json を Write ツールで保存する
4. 残りの蓄積リスト全件を Step 5 と同じ形式で表示する
   - No. は振り直さず、元の番号を維持する（歯抜けになってもよい）

---

## 表示モード: Step 7

ユーザーが「UIを開いて」「ボタンで操作したい」「一覧を開いて」等と指示した場合:

1. `mcp__Claude_Preview__preview_start` で name=`未返信メール UI` を指定してサーバーを起動する
   - launch.json に定義済み（port: 8766、server.py を Python で実行）
   - serverId が返ってくる
2. `mcp__Claude_Preview__preview_screenshot` で表示を確認する
3. 「Claude Code 内のプレビューパネルに一覧を表示しました」と伝える
4. No.X のリンクをクリックするとGmailが開くこと、「✕ 返信不要」ボタンで削除できることを案内する

**注意**: 同じ会話内で既に起動済みの場合は reused: true になりサーバーIDも使い回せる。

---

## 注意事項

- 同一スレッドで複数メッセージがある場合は、最新のメッセージの状態で判断する
- 並列処理: スレッド読み込みは可能な限り並列で実行して高速化する
- state.json は Read → 更新 → Write の順で必ず保存すること（保存し忘れると次回実行時に重複する）
