# Gmail返信スキル

メールの返信文を作成し、Gmailから「全員に返信」で送信するスキル。

## ワークフロー

### 1. 元メールの確認

ユーザーが返信したいメールを特定する。以下のいずれかの方法で元メールを取得する:

- ユーザーがスレッドIDやメッセージIDを指定した場合: `gmail_read_thread` または `gmail_read_message` で内容を取得
- ユーザーが検索条件を指定した場合: `gmail_search_messages` で検索してから内容を取得
- ユーザーが直接メール内容を貼り付けた場合: その内容を使用

### 2. 返信文の作成

以下のフォーマットに**必ず**従うこと:

```
○○さま
稲垣です

（本文）

（引用文全文）
```

#### フォーマットの詳細ルール

- **宛名**: 元メールの差出人の名前を使い「○○さま」とする。姓のみでよい。名前が不明な場合はメールアドレスのローカル部分を使う
- **自己紹介**: 必ず「稲垣です」を2行目に書く
- **本文**: 空行を1行空けてから本文を書く。ユーザーの指示に基づいて適切な返信内容を作成する。ビジネスメールとして自然な日本語で書く
- **引用文**: 本文の後に空行を1行空け、元メールの全文をそのまま引用する。各行の先頭に `> ` を付ける。引用は元メールの全文を含めること（件名、日付、差出人、本文すべて）

#### 引用文のフォーマット例

```
> On 2026/03/28 14:30, 田中太郎 <tanaka@example.com> wrote:
>
> お疲れ様です。
> 田中です。
>
> 明日の会議の件ですが、14時からでよろしいでしょうか。
> よろしくお願いいたします。
```

### 3. ユーザーへの確認

返信文の全文をユーザーに表示し、以下を確認する:

- 返信内容が意図通りか
- 宛先（全員に返信の宛先リスト）が正しいか
- 修正が必要な箇所はないか

**ユーザーの明示的な承認を得るまで絶対に送信しない。**

### 4. 送信

ユーザーの承認後、`gmail_create_draft` を使用してドラフトを作成する。

- `threadId` を指定して元のスレッドへの返信とする
- 宛先は**全員に返信（Reply All）**: 元メールの From, To, Cc すべてを含める（自分自身のアドレスは除く）
  - 元メールの From → To に設定
  - 元メールの To と Cc から自分を除いた全員 → Cc に設定
- `contentType` は `text/plain` を使用

ドラフト作成後、ユーザーに送信の最終確認を行い、承認を得てから Gmail UI での送信を案内する（Gmail API のドラフト送信機能がある場合はそれを使用）。

## 重要な注意事項

- 返信フォーマットは厳守すること。「○○さま」「稲垣です」の形式は変更不可
- 全員に返信が基本。ユーザーが明示的に個別返信を指定した場合のみ例外とする
- 引用文は元メールの全文を含めること。省略しない
- ユーザーの承認なしにメールを送信・作成しないこと
